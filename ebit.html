Token Mechanics
5.1

Immutable Supply

EBIT uses a fixed, immutable supply model: minted once, transferred to Treasury, issuer blackholed — no future minting, no inflation, no discretionary supply changes. Ever.

5.2

Treasury Mechanics

The Treasury wallet performs liquidity provisioning, controlled burns, supply rebalancing, and AMM deposits/withdrawals. Actions are data-driven, triggered by XRPL AMM metrics, Bitcoin mempool data, Cosmos chain state, Stellar liquidity, Chainlink oracles, Graph analytics, and Arbitrum DeFi signals.

5.3

AMM Mechanics

The AMM provides algorithmic pricing, automatic fee generation, liquidity rewards, and market-driven behavior. Backend systems monitor pool depth, fee generation, arbitrage windows, volatility, and liquidity imbalances across chains including Arbitrum.

5.4 / 5.5

Owner & Hot Wallet

The Owner/Admin wallet manages governance and system oversight — it does not distribute dividends or profit guarantees. The hot wallet automates AMM rebalancing, reward harvesting, and ISO-aligned telemetry under non-discretionary logic.

6. ISO-20022 Event Model

EBIT produces structured events for token creation, transfers, treasury operations, burns, liquidity provisioning, AMM creation, adjustments, rewards, automation triggers, system health, parameter updates, and authorization changes. Events include timestamps, wallet addresses, amounts, asset types, data sources, trigger conditions, and system state — fully aligned with the global financial messaging standard.
